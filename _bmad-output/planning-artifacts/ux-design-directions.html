3 · SetupCardStack — pattern detail

The signature component. Treats three SKUs (deck + trucks + wheels) as one buyable unit with: shared border, unified header strip, compatibility-validated chip, three sub-cards visually unified, why-this rationale, single accent CTA at the bottom.

Anatomy

Header strip · setup name, total price (mono), Setup-validated chip (`grounded` token).
Sub-cards · deck · trucks · wheels — each with hero image, product name, sticker tags, mono price.
Rationale · 1–2 sentence why-this in skater vernacular; trade-offs called out.
CTA · single accent button. Primary action: "Add complete setup · $XXX.XX". Secondary action ("View setup details") only on alternative recommendations.

Mobile vs. Desktop

Same component, two layouts. Mobile stacks the sub-cards vertically; desktop renders them horizontally in a 3-column grid. Touch targets remain ≥ 44 × 44 px on both.

4 · Streaming behavior — skeleton → token-fill → final

When the user sends a message, an acknowledgement token streams in (≤ 1 s), then a skeleton recommendation card appears with real card framing. As tokens arrive, the rationale text fills in left-to-right while the rest of the card hydrates. Animation respects prefers-reduced-motion.

5 · ConfidenceBoundaryDisclosure (Diego's case)

When the LLM has matched on some attributes but the catalog is missing others, the system says so calmly. Visually distinct from a generic error: uncertain token (amber), left accent line, calm copy. The recommendation still ships — this is *honest framing*, not a failure state.

Bushings · medium-soft, 88a–90a band $11.99–$24.00

Partial match

I matched on the size (Indy 149) and medium-soft durometer band you described. Our catalog doesn't have an explicit "downhill" tag for these yet, so I can't be 100% sure they handle high-speed dive the way you want — but the durometer and shape are correct for your trucks.

7 · DegradedModePanel (LLM circuit-breaker tripped)

When the LLM provider is unavailable or sustained tail latency exceeds threshold, the assistant gracefully falls back to the existing keyword-search experience. Critical UX: framed as confident, not failed. The user's last query is pre-filled. No alarm icons, no error tones.
